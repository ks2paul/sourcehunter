# SourceHunter MVP 范围 V1.0

## 1. MVP 原则

V1 的核心原则是准确供应商检索。

优先级顺序：

1. 数据准确
2. 供应商链接准确
3. 价格和 MOQ 提取准确
4. 供应商去重可靠
5. AI 推荐解释清晰
6. UI 简洁可用

如果数据无法可靠获取，系统应返回少量可信结果，而不是返回完整但不可信的结果。

## 2. 必须实现

### 产品输入

- 产品关键词输入
- 可选目标价格
- 可选 MOQ 偏好
- 可选供应商偏好
- 可选产品图片上传

### AI 关键词扩展

- 英文采购关键词
- 中文采购关键词
- 产品变体词
- 低置信度词不得自动用于搜索，除非标记为辅助词

### 平台搜索

- 1688 搜索
- Made-in-China 搜索
- Playwright 浏览器自动化
- 搜索结果列表抓取
- 产品详情页抓取
- 供应商主页或公司页抓取

### 数据提取

- 公司名称
- 平台
- 供应商链接
- 产品名称
- 产品链接
- 价格
- MOQ
- 地区
- 年限
- 可公开获取的联系方式

### 去重

- 基于公司名、地址、电话、网站、店铺链接的去重
- 输出唯一供应商
- 每个供应商有不可变 `supplier_id`

### 评分

- 综合评分
- 分项评分
- 推荐理由
- 推荐动作

### 前端

- 搜索输入页
- 搜索进度状态
- 结果列表
- 供应商详情
- 筛选和排序
- 原始链接打开

## 3. 不在 MVP 范围内

- 自动登录平台账号
- 自动发送询盘
- CRM 管理
- 批量项目管理
- 供应商聊天记录
- 采购订单
- 付款流程
- 合同管理
- 高级数据看板
- 多用户权限系统

## 4. 质量要求

- 不显示虚假数据
- 不使用 mock supplier 作为真实结果
- 所有供应商必须有来源平台和原始 URL
- 价格必须来自页面提取
- 联系方式必须来自公开页面提取
- 缺失字段必须明确显示 unavailable
- 抓取失败必须保留错误原因

## 5. 验收标准

使用 3 个测试产品进行验收：

- Handheld Fan
- Pet Wipes
- Picture Frame

每个测试产品应验证：

- 是否能生成中英文搜索词
- 是否能成功访问目标平台
- 是否能提取真实产品列表
- 是否能进入详情页
- 是否能识别供应商
- 是否能去重
- 是否能返回最多 5 个唯一供应商
- 点击结果是否打开正确原始链接
